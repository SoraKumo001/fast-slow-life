# Fast Slow Life

村を発展させながらダンジョンを攻略していく、テキストベースの街経営＆ダンジョンRPGシミュレーションゲームです。
本リポジトリは、ゲームエンジン **Godot** での実装を想定したゲームデザイン原案（仕様書・パラメータ設計）を格納しています。

---

## 💡 コアシステム

### 1. 施設での素材加工とアップグレード
村の施設は単なる売買や強化の場所ではありません。
採取してきた一次素材（食料、鉱石、薬草など）を施設で加工し、より価値の高い中間素材や装備品、薬品を「クラフト」できます。
また、集めた素材を消費して施設自体の機能を「アップグレード」することで、新たなレシピの解放や生産効率の強化が行えます。

### 2. 職業適性に応じた村人の自動行動
プレイヤーは村人一人ひとりに対し、「採取」か「討伐」かの大まかな方針（指示）を出します。
村人は与えられた指示に対し、自身の職業（農民、鉱夫、猟師、戦士など）の適性やステータスに基づいて、自動的に「最も適した具体的な行動（どの素材を採取するか、どの敵を討伐するか）」を決定して実行します。

### 3. ループする期限と目標ボス
ゲーム開始時、プレイヤーには「目標日数」と「討伐すべきボス」が提示されます。
期限内にボスを倒すと、ゲームは終了せず、さらに手強い「次の目標ボス」と「新たな期限」が設定されて継続します。
もし期限内にボスを倒せなかった場合はゲームオーバーとなり、それまでの発展度に応じた「ソウルポイント」を獲得できます。これを使用して永続的なバフを獲得し、次の周回（強くてニューゲーム）へと挑むことができます。

## 🛠️ 基本仕様（概要）

- **グラフィック**：基本的にテキストおよびシンプルなUIベース（Godotの Control ノードを活用）。
- **ゲームサイクル**：ターン制（1日経過ボタン、またはオート進行）。
- **主要リソース**：
  - ゴールド（資金）
  - 食料（毎日、村人の人数に応じて消費。枯渇するとペナルティ）
  - ソウルポイント（周回用資源）
