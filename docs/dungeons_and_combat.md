# ダンジョン＆戦闘設計書：エリアと戦闘ロジック

ゲームに登場するフィールド・ダンジョンのデータ設計、探索率の進行仕様、および戦闘におけるダメージ計算や報酬決定ロジックの設計です。

---

## 1. フィールド・ダンジョン（エリア）設計

各エリアには「探索度（0%〜100%）」があり、探索度の上昇に伴い、よりレアな素材の発見や強力なモンスターとの遭遇が発生します。

### 探索率の進行

- 毎時間、エリアに派遣された村人のステータスに応じて探索度（%）が上昇します。
- **1時間あたりの探索度上昇量**:
  $$\text{探索上昇量(\%)} = \frac{\text{村人のDEX} \times 0.2 + \text{村人のAGI} \times 0.2}{\text{エリアの探索難易度} \times 24.0}$$
- **要素の解放**:
  - 探索度が以下の閾値に達すると新しい要素が解放されます：
    - 40%: 第2段階の素材・モンスター解放
    - 50%: 第3段階（該当する場合）
    - 70%: 最終段階（該当する場合）
  - 各スポット・モンスターは個別の `unlockedAtProgress` を持ちます。
- **ボス討伐の条件**: 探索度が100%になるとボス挑戦が可能になります。

### 採取・討伐の進捗ゲージとリスポーン

各素材スポットおよびモンスターは、「進捗ゲージ（0%〜100%）」を個別に持ちます。

- **進捗ゲージの上昇**: 村人が同エリアで採取・討伐行動を行うたびに上昇。
  - **採取ゲージ上昇式**: `(DEX × 0.8 + 10) / 採取難易度`
  - **討伐ゲージ上昇式**: `(AGI × 0.8 + 10) / (モンスターレベル × 0.4 + 2)`
- **アイテム取得・戦闘発生**: 進捗ゲージが100%に達した時点で、採取ならアイテム獲得、討伐なら戦闘が発生。
- **リスポーン待機時間**:
  - 一般素材・ゴブリン等: 3時間
  - 石材・銅鉱石・鉄鉱石・銀鉱石・水晶の欠片等: 6時間
  - 魔法石: 12時間
  - 中級モンスター（オーク、ワーウルフ等）: 8時間
  - ボス級モンスター: 16時間

## 1.5 脅威度とお布施システム

旧来の「日数制限」によるゲームオーバーを撤廃し、**ダンジョンごとの脅威度（Threat Level）** を時間圧の主軸とする。

### 脅威度の上昇

- 各ダンジョンは `threatLevel: 0-100` を持つ
- **未攻略ダンジョン**（`unlockedAtTier === currentTier` かつ `bossDefeated === false`）のみ、毎時加速度的に上昇
- 攻略済みダンジョンは 0% に固定
- 上昇式: `累積(t) = BASE × t ^ EXPONENT`（デフォルト `BASE=0.1, EXPONENT=1.1`）
- Tier1 開始から **約 14 ゲーム日** で 100% に到達し**即ゲームオーバー**

### お布施（オファリング）

プレイヤーが任意のタイミングで `offering(dungeonId, percentToReduce)` を実行できる。

- 軽減量 X% に応じてゴールドを消費
- コスト計算式（指数逓増）: `cost(X) = ceil(BASE(tier) × (ESC^X − 1) / (ESC − 1))`
  - デフォルト `OFFERING_BASE_GOLD_BY_TIER = [0, 2, 10, 50, 250, 1000]` (Tier別), `ESC = 1.08`
- 例 (Tier1): 10% → 29 G, 30% → 227 G, 50% → 1,148 G, 90% → 25,448 G
- **全額リセットはお布施では割に合わない** ように設計されており、最終的な解決手段は依然としてボス討伐

> 詳細な実装値・Tier別表は [`docs/system_design.md` §11.4](./system_design.md#114-お布施オファリング) を参照。

### 襲来イベントについて

> **変更点**: 旧仕様の「襲来イベント（猶予6時間・被害発生・3回累積でゲームオーバー）」は
> 撤廃済み。実装上は **脅威度が 100% に達した時点で即ゲームオーバー** となる。
> 詳細は [`docs/system_design.md` §11.5](./system_design.md#115-ゲームオーバー脅威度max到達) を参照。

### ボス討伐時のリセット

エリアボスを討伐すると**全ダンジョンの脅威度が 0% にリセット**される。新しい Tier のエリアが解放され、再び脅威度 0% からゲームが進行する。

### 詳細な定数・実装

| 項目               | 値 / 場所                                                         |
| ------------------ | ----------------------------------------------------------------- |
| 基本上昇率         | `THREAT_BASE_RATE_PER_HOUR = 0.1`                                 |
| 加速係数           | `THREAT_ACCELERATION_EXPONENT = 1.1`                              |
| ゲームオーバー閾値 | `THREAT_MAX = 100`                                                |
| お布施基本コスト   | `OFFERING_BASE_GOLD_BY_TIER = [0, 2, 10, 50, 250, 1000]` (Tier別) |
| お布施逓増係数     | `OFFERING_ESCALATION_FACTOR = 1.08`                               |

仕様の詳細と実装計画は [`docs/system_design.md` §11](./system_design.md#11-ダンジョン脅威度システム) および [`.omo/plans/dungeon-threat-system.md`](../.omo/plans/dungeon-threat-system.md) を参照。

## 2. 戦闘システムとダメージ計算式

戦闘はテキストログ形式の自動ターン制バトルです。通常戦闘は最大10ターン、ボス戦は1時間に5ラウンド実行され、敵のHPを0にすれば勝利、村人のHPが0になるかターン超過で敗北（一時撤退）となります。

### ① 命中・回避判定

$$\text{命中率(\%)} = 85 + (\text{アタッカーのDEX} - \text{ディフェンダーのAGI}) \times 1.5$$

最低命中率：50%、最高命中率：100%

### ② クリティカルヒット

$$\text{クリティカル確率(\%)} = \text{DEX} \times 0.1\% \text{（上限 30\%）}$$

クリティカル時はダメージが **1.5倍** になり、防御力を 50% 無視。

### ③ ダメージ計算式

#### 物理攻撃（戦士、猟師、農民、鉱夫など）

$$\text{与ダメージ} = \max\left(2, (\text{STR} \times 1.5 + \text{武器攻撃力}) - (\text{敵の物理防御力} + \text{敵のVIT} \times 0.5) \times \text{効率} \times \text{職業ボーナス}\right)$$

- 戦士の職業ボーナス: 1.3倍
- 効率には飢餓ペナルティ(0.5)やスタミナ0ペナルティ(0.3)が反映

#### 魔法攻撃（魔術師、僧侶、薬師など）

$$\text{与ダメージ} = \max\left(2, (\text{INT} \times 1.8 + \text{武器知力補正}) - (\text{敵の魔法防御力} + \text{敵のINT} \times 0.5) \times \text{効率}\right)$$

### ④ ツケ（未払い）ペナルティ

所持金がマイナスの村人は全ステータスが `0.7倍` に低下します（`applySalaryDebuff`）。

### ⑤ 最小ダメージ

- 通常戦闘: 10（`MIN_DAMAGE`）
- ボス戦: 30（`MIN_BOSS_DAMAGE`）

## 3. 報酬システム (経験値とアイテムドロップ)

### 獲得経験値

```typescript
const eduBonus = 1.0 + soulUpgrades.education * 0.1; // 最大1.5倍
const expGained = Math.floor(monster.expReward * eduBonus);
```

- 英才教育のソウルアップグレード1Lvにつき +10%（最大5Lvで+50%）。
- レベル差による補正は**ありません**（コード上は一律で基本EXP × 教育ボーナス）。

### ドロップアイテム判定

各モンスターは固有のドロップテーブルを持ちます。

```typescript
const hunterBonus = v.currentJob === "猟師" ? 1.5 : 1.0;
if (Math.random() < drop.chance * hunterBonus) {
  /* ドロップ */
}
```

- 猟師の場合、ドロップ確率が **1.5倍** になります。

## 4. 回復薬・スタミナポーションの自動使用システム

### 配布ルール（派遣時）

村人がダンジョンへ自動派遣される際、倉庫から以下のアイテムを所持金で自動購入します：

1. **回復薬**: 最大2個まで。優先順位は効果量の高い順：`elixir > high_potion > mid_potion > potion`
2. **スタミナポーション**: 最大2個まで。倉庫にある場合自動購入。

### 自動使用条件

| アイテム           | 使用条件                            | タイミング                                                   |
| :----------------- | :---------------------------------- | :----------------------------------------------------------- |
| 回復薬             | HPが最大HPの50%以下、かつ所持数 > 0 | 探索行動前、通常戦闘中各ターン開始時、ボス戦中各ターン開始時 |
| スタミナポーション | スタミナが30以下、かつ所持数 > 0    | 毎時間の活動開始時                                           |

### 返却ルール

村人が帰還を完了した際、未使用の回復薬・スタミナポーションはすべて倉庫に返却されます。

## 5. ボス討伐システム

### ① 戦闘開始

- ダンジョンの探索度が100%に達するとボス挑戦が可能に。
- プレイヤーが最大4人の村人を選択して派遣。
- ボス戦は自動ターン制で進行。

### ② 戦闘進行

- 1時間ごとに `BOSS_BATTLE_ROUNDS(5)` ラウンドの戦闘が実行。
- **各ラウンドの処理順**:
  1. ボスのHPが `BOSS_REGEN_PERCENT(0.2%)` 回復（最大HPを超えない）
  2. 参加村人全員の行動（ポーション使用 → 僧侶ヒール判定 → 攻撃）
  3. ボスの反撃（生存中のアタッカーからランダムに1体攻撃）
- ヒール条件: 僧侶がパーティの中で最もHP割合が低いメンバー（50%以下）を回復。

### ③ 撃破時

- ボスHPが0になると撃破。参加村人全員に `monster.expReward × educationBonus` の経験値。
- Tierアップ: 次のTierに移行し、新たな制限日数が設定。
- 参加村人は自動的に待機状態に戻る。

### ④ 撤退・全滅

- 参加村人が全員戦闘不能になるとボス戦は自動終了。
- 戦闘不能の村人は強制的に帰還（休息状態へ）。
