# システム設計書：ゲームサイクル・目標・周回

本作「Fast Slow Life」の根幹となる、時間経過ルール、目標更新、および周回（強くてニューゲーム）システムの詳細設計です。

---

## 1. 時間経過と1時間ごとの処理フロー

ゲームは「1時間」を最小単位として進行します。時間の自動進行中、または手動で「1時間進める」操作が行われるごとに、裏側で以下のステップが順番に処理されます。

### 処理フロー（TypeScript/Zustand実装イメージ）

```typescript
// useGameStore (Zustand) の一部として実装するイメージ
const advanceHour = () => {
  const state = get();
  if (state.gameOver) return;

  // 1時間進行させる計算（calculateAdvanceHour）
  const result = calculateAdvanceHour(state);

  // 状態のセット
  set({
    currentDay: result.currentDay,
    currentHour: result.currentHour,
    villagers: result.villagers,
    facilities: result.facilities,
    dungeons: result.dungeons,
    inventory: result.inventory,
    // (中略)
  });

  // 自動装備と自動派遣の実行
  get().autoEquipAll();
  get().dispatchIdleVillagers();
};
```

### 各ステップの詳細

#### ① 食料の消費と飢餓

- **消費量**：1時間ごとに `消費食料 = 村人の総数 * (1.0 / 24.0)` が倉庫から消費されます（内部的には小数のまま管理し、画面上は整数に丸めて表示）。
- **食料不足（飢餓）ペナルティ**：
  - 倉庫の食料が 0 の状態で1時間が経過する（または消費分が不足する）と、村人全員に「飢餓」デバフが付与されます。
  - **飢餓効果**：全村人の最大HPの 0.4% が減少（24時間で約10%減少に相当。毎時間累積）、および採取・討伐・クラフト時の作業効率（ステータス効果）が 50% に低下します。
  - 食料が再び倉庫に補充されると、飢餓状態は即座に解除され、HP減少は停止します。

#### ② 施設のクラフト・アップグレード進捗処理

- **生産進捗**: クラフト中のアイテムは「所要時間（時間単位）」を持っており、1時間経過するごとに残り時間が 1 減少します。残り時間が 0 になると生産が完了し、成果物が倉庫に追加され、クラフトに従事していた村人が解放されます。
- **アップグレード進捗**: 施設アップグレードも同様に「所要時間」が1時間ごとに減少し、0になると施設レベルが上昇します。

#### ③ 村人の行動処理

- 詳細は「キャラクター＆職業設計書」で定義されますが、1時間ごとに以下の処理を行います。
  - **スタミナ消費**: 「採取」や「討伐」で活動中の村人は、1時間ごとにスタミナが 5 減少します。スタミナが 0 になると、行動効率が著しく低下するか、強制的に「休息（帰還）」状態になります。
  - **進捗とイベント**: 派遣先のダンジョンへの「往路」「復路」にそれぞれ数時間（ダンジョンの距離に依存）要します。現地到着後、1時間ごとに「採取判定」または「敵との戦闘判定」が発生し、アイテム獲得や経験値取得が行われます。

#### ④ HP・スタミナの回復（宿屋・休息処理）

- 「休息」状態の村人は、1時間ごとにHPとスタミナが回復します。
- **回復量**: 宿屋のレベルに依存します（例: 宿屋Lv.1で1時間あたり HP+10, スタミナ+15）。

## 2. 目標と期限のループシステム

ゲームは一回ボスを倒したら終わりではなく、目標を達成するたびに「次の目標」と「新たな期限」が設定されて継続します。

### 目標スケジュール定義

| 段階 (Tier) | 目標ボス               | 制限日数 (累計) | 解放される要素                           |
| :---------- | :--------------------- | :-------------- | :--------------------------------------- |
| **Tier 1**  | ゴブリンロード         | 30日目まで      | ダンジョン「廃鉱山」、施設「鍛冶屋」     |
| **Tier 2**  | ゴーレム               | 70日目まで      | ダンジョン「魔獣の谷」、施設「錬金工房」 |
| **Tier 3**  | キマイラ               | 120日目まで     | ダンジョン「世界樹の根」、施設「交易所」 |
| **Tier 4**  | アークデーモン         | 180日目まで     | ダンジョン「深淵の奈落」                 |
| **Tier 5**  | 終焉の竜（ラストボス） | 250日目まで     | エンディング（以降は無限サバイバルへ）   |

### 目標クリアと期限更新処理

- プレイヤーがボスを撃破した時点で「目標達成」となり、ただちに次の目標がアクティブになります。
- **残り日数の猶予**：早くボスを倒すほど、次のボス討伐までの準備期間（猶予日数）が多くなります。
  - 例：15日でゴブリンロード（期限30日）を倒した場合、次のゴーレム（期限70日）まで「残り55日」の猶予が生まれます。

## 3. ゲームオーバーと周回要素 (強くてニューゲーム)

### ゲームオーバー条件

1. **期限切れ**：アクティブな目標ボスの制限日数に達した時点で、ボスが未討伐の場合。
2. **全滅**：村人が全員死亡し、かつ新しい村人を雇うゴールドが尽きた場合。

### ソウルポイント (SP) 計算式

ゲームオーバー時、プレイヤーの進行度や資産に応じて「ソウルポイント (SP)」が算出されます。

$$SP = \lfloor \frac{\text{所持ゴールド}}{1000} \rfloor + \lfloor \frac{\text{倉庫内全アイテムの売値合計}}{100} \rfloor + (\text{撃破した中ボス・ボス数} \times 50) + (\text{到達日数} \times 2)$$

### 周回強化ショップ (転生バフ)

獲得したSPを消費して、次回プレイ時に永続適用されるバフを購入できます。

| バフ名           | 効果                                | 1レベルあたりコスト | 最大レベル |
| :--------------- | :---------------------------------- | :-----------------: | :--------: |
| **先祖の遺産**   | 初期ゴールド +500                   |        10 SP        |     10     |
| **豊かな備蓄**   | 初期食料 +100                       |        5 SP         |     10     |
| **英才教育**     | 村人の獲得経験値 +10%               |        20 SP        |     5      |
| **頑強な肉体**   | 全村人の初期ステータス（全属性） +2 |        15 SP        |     5      |
| **効率的な建築** | 施設アップグレードの素材要求量 -5%  |        25 SP        |     5      |
| **値切り上手**   | 転職に必要なゴールド -10%           |        15 SP        |     5      |

### 引き継がれる要素

- 周回ショップで解放した「永続バフ」
- 過去の周回数、最高到達日数などの実績データ

### リセットされる要素

- 村人のレベル、ステータス、装備、職業（初期状態の村人3名から再スタート）
- 施設のレベル（すべて初期状態）
- 所持ゴールド、食料、素材アイテム、装備
- ダンジョンの探索率、ボス撃破状況

## 4. 目標設定と自動化システム (生産・行動の自動化)

プレイヤーがアイテムごとに目標個数を設定し、それに基づいて村人の行動や施設のクラフトを自動化するシステムの設計です。

### ① UIでの目標設定と表示

- **アイテムリストの解放前非表示 (Visibility Filter) [NEW]**:
  - 倉庫アイテムリストは、現在の進行度（Tier）や建設された施設のレベルに応じて、まだ製造・入手することができない未来のアイテムを非表示にします。
  - 例：ダンジョン「廃鉱山」が解放されておらず「鉄鉱石」が入手可能状態（所持数0かつ採取・ドロップ不可）になっていない場合は、加工工房が解放されていても「鉄のインゴット」はリストに表示されません。
- **目標個数の設定**:
  - アイテムリストの各項目、および**アイテム詳細ダイアログ**で「目標個数 (Target Amount)」の設定が可能です。
  - 詳細ダイアログ上では、目標数を直感的に設定できる `+1`, `+10`, `+100`, `リセット` の調整ボタンが使用できます。

### ② 目標に基づく村人の派遣自動化

- **即時派遣 [NEW]**:
  - プレイヤーが目標個数を設定・変更した瞬間、または新しい村人を雇用した瞬間に、時間経過（1時間進行）を待つことなく、待機中（かつ方針が休息でない）の村人が即座に自動アサインされてダンジョンへ派遣されます。
- **採取ターゲットの自動選定スコア（達成率優先） [NEW]**:
  - 自動採取（個別ターゲットが指定されていない状態）の意思決定スコア計算時、目標が設定されていないアイテムにはペナルティ（スコア0.1倍）、目標達成済みのアイテムにはより強いペナルティ（スコア0.01倍）を適用します。
  - **目標未達アイテムの優先強化**: 目標数に対する現在の達成率が低いほど高いスコアを付与します（重みは `2.0 - (現在数 / 目標数)`）。達成率0%なら2.0倍、50%なら1.5倍、99%なら約1.01倍と滑らかに変動します。
- **討伐ターゲットの自動選定（達成率優先） [NEW]**:
  - 自動討伐時にモンスターを選択する際、そのモンスターのドロップアイテムの中に「目標未達のもの」があるモンスターを優先ターゲットにします。
  - 目標未達ドロップを持つモンスターが複数いる場合、**達成率が最も低い（最も不足している）ドロップアイテムを持つモンスター**を優先して選択します。
- **自動派遣の優先度（達成率優先） [NEW]**:
  - 待機村人を自動派遣する際、職業の得意カテゴリで絞り込んだ後、その中で達成率が最も低いアイテムが採取できるエリアへ優先して派遣します。
- **エリアの全目標達成による自動帰還 [NEW]**:
  - 自動派遣された村人は、そのダンジョンで採取・討伐可能なすべてのターゲットの目標値が達成された場合、自動的に村への帰還（休息・待機状態への遷移）を開始します。帰還後に待機状態となった村人は、再び不足アイテムがある別のエリアへと自動派遣されます。
- **初期アサイン方針の最適化 [NEW]**:
  - 新規雇用時およびゲーム開始時の村人の初期方針を「採取 (gather)」とし、待機状態になった際シレスに自動派遣ループに乗るように調整されています。

### ③ 目標に基づく施設の自動クラフト

- **自動加工・クラフトの開始**:
  - 設定された中間素材（木板、鉄インゴット等）や消費アイテム（回復薬等）、装備品の在庫が目標個数に達していない場合、倉庫に必要な一次素材（木材、鉄鉱石等）が揃っているかをチェックします。
  - 素材条件を満たしており、かつ対応する施設（工房、鍛冶屋、錬金工房）がアイドル状態であれば、プレイヤーの手動指示なしで自動的にクラフトキューに仕事が追加され、生産が開始されます。
- **職人の自動アサイン**:
  - 施設での自動クラフト開始時、手が空いている職人（crafter）がいる場合、自動的にその職人を加工担当として稼働させ、生産速度を最大化します。

## 5. 周回・ゲームオーバーの挙動詳細 [NEW]

### 転生後の自動ゲーム開始

- **転生ボタン（「バフを適用して新しい周回を開始」）を押してゲームをリセットした場合**、初期化後にゲームが**自動的に一時停止なしで開始**します。
- 手動リセット（ゲームオーバー以外のルートからリセット）の場合は従来通り一時停止（`isPaused: true`）状態で開始します。
- 転生時は `activeBoss`（前の周回のボス戦状態）も必ず `null` にリセットされるため、前の周回の戦闘状態が引き継がれることはありません。

## 6. データ永続化とマスタ同期システム [NEW]

本作は Zustand の `persist` を用いて localStorage にセーブデータを保存しますが、コードの更新をスムーズに反映させるため、ロード時に以下のマージ処理が自動実行されます。

1. **ダンジョン情報の同期**:
   - セーブデータからは、プレイヤーの進行状況である「探索度 (`explorationProgress`)」のみを復元します。
   - それ以外のパラメータ（ダンジョン名、移動距離、推奨レベル、解放される採取素材や難易度、出現するモンスターなど）は、常にコード上の最新のマスタデータ定義で動的に上書き同期されます。
2. **未建設施設のコスト同期**:
   - 施設がまだアンロックされていない（`level === 0`）場合、そのアンロックに必要なゴールドコストや素材リストは、コード上で変更された最新のマスタデータ定義で上書きされます。これにより、交易所などのアンロック条件（初期ゴールド不要化など）の変更がセーブデータロード時に即時適用されます。
3. **村人の新規フィールド（`potionCount`）の同期**:
   - 古いセーブデータには `potionCount` フィールドが存在しない場合があるため、ロード時にデフォルト値 `0` を補完します。
