# システム設計書：ゲームサイクル・目標・周回

本作「Fast Slow Life」の根幹となる、時間経過ルール、目標更新、および周回（強くてニューゲーム）システムの詳細設計です。

---

## 1. 時間経過と1時間ごとの処理フロー

ゲームは「1時間」を最小単位として進行します。`advanceHour` が呼ばれるたびに、以下のステップが順番に処理されます。

### 処理フロー（calculateAdvanceHour の実装順序）

1. **時間進行**: `currentHour += 1`。24時間経過で日付更新（`currentDay += 1`, `currentHour = 0`）。
2. **日次イベント処理**:
   - 所持金マイナス状態の追跡（`consecutiveNegativeGoldDays`）。3日連続で破産により**ゲームオーバー**。
   - **マーケットトレンド**の生成：アンロックされた街からランダムに1つ選び、需要アイテムの価格が本日限定で上昇。
3. **脅威度進行**: 未攻略ダンジョンの脅威度を加速度的に上昇（詳細は [§11. ダンジョン脅威度システム](#11-ダンジョン脅威度システム) 参照）。
4. **ゲームオーバー判定（脅威度MAX）**: 脅威度が100%に達したダンジョンがある場合、即ゲームオーバー。
5. **リスポーン更新**: 全ダンジョンの採取・モンスターのリスポーン残り時間を1時間減少。
6. **食料消費処理**: 各村人が `1.0/24.0` 単位の食料を消費。食料が足りない村人は飢餓状態に。
7. **食料代徴収（日次）**: 各村人の所持金から食料代（2 G）を引き落とし、プレイヤーに支払われる。料理を食べている村人は料理の基本価格分を支払う。
8. **食料Buff適用ログ**: 調理された料理を食べている村人の一覧をログに出力。
9. **探索度進行**: 各ダンジョンで活動中の村人のステータスに応じて探索度が上昇。
10. **クラフト・アップグレード進行**: 施設の生産進捗とアップグレード進捗を1時間分進行。
11. **資源施設生産（12時間ごと）**: 農場・伐採所・採石場が12時間ごとに素材を自動生産。
12. **ボス戦処理**: アクティブなボス戦の1ラウンド分を処理（HP回復、攻撃、反撃）。
13. **村人活動処理**: 各村人の状態に応じた行動実行（休息、移動、採取、討伐）。
14. **自動クラフト**: 目標個数に基づき、不足アイテムの自動生産開始。
15. **交易馬車進行**: 派遣中の馬車の残り時間を減少。帰着時は自動回収。
16. **街のアンロック**: 現在のTierに応じて交易先の街を解放。
17. **自動取引**: 取引ルールに基づく超過在庫の自動輸出。
18. **プール自動購入**: 村人の仮置き場にあるアイテムを自動買取。
19. **ツケ状態更新**: 村人の所持金がマイナスかどうかで未払いフラグ更新。

> **注**: 旧仕様の「`currentDay > gameLimitDays` による日数ゲームオーバー」は **撤廃** 済み。`gameLimitDays` フィールドは型互換性のため残置するが、UI には表示されない。

### ゲームオーバー条件

1. **脅威度MAX到達**: 未攻略ダンジョンの脅威度が100%に達した場合（詳細は §11.5）。
2. **破産**: プレイヤーの所持金がマイナスの状態が3日間続いた場合。

## 2. 食料消費と飢餓システム

### 食料消費の仕組み

- 各村人は1時間ごとに `FOOD_CONSUMPTION_PER_VILLAGER(1.0/24.0)` 単位の食料を消費します。
- 消費の優先順位：
  1. 高級料理（竜鱗の贅沢鍋、魔獣のローストなど）→ 村人のステータス強化効果あり
  2. 調理食料（パン、干し肉、サラダなど）→ ステータス強化効果あり
  3. 生の食材（生肉、野菜、小麦）
- 村人が食べた食料が調理済み料理の場合、`activeFoodBuffId` にセットされ、1時間ごとにステータスボーナスが適用されます。
- **貧困村人ルール**: 所持金が200G未満の村人は、安価な食料（生の食材 → 基本料理）を優先的に消費します。

### 飢餓（食料不足）

- 倉庫に消費可能な食料が一切ない場合、その村人は**飢餓状態**（`isStarving = true`）になります。
- **飢餓効果**:
  - 毎時、最大HPの `STARVATION_HP_LOSS_PERCENT(0.4%)` が減少
  - 採取・討伐時の効率が `STARVATION_EFFICIENCY_PENALTY(0.5)` に低下

## 3. 目標とTier進行のループシステム

### Tier進行スケジュール

| 段階 (Tier) | 目標ボス         | 解放される要素                               |
| :---------- | :--------------- | :------------------------------------------- |
| **Tier 1**  | ゴブリンロード   | 廃鉱山、加工工房、調理場、武器屋、交易所     |
| **Tier 2**  | アイアンゴーレム | 魔獣の谷、錬金工房、街「港町アイアンポート」 |
| **Tier 3**  | キマイラ         | 世界樹の根、街「魔法都市マギカ」             |

| **Tier 5**  | 終焉の竜         | エンディング                                 |

1. エリアボス討伐成功 → `currentTier += 1`
2. 全ダンジョンの脅威度が 0% にリセットされる（[§11.6](#116-ボス討伐時のリセット) を参照）
3. 解放済みダンジョン一覧が新 Tier のエリアを含むよう更新される
4. Tier 5 ボスを討伐するとゲームクリア（エンディング）

## 4. 村人の経済システム

### 個人所持金（gold）

各村人は個人の所持金を持ちます。

- **収入源**:
  - 採取・討伐で得たアイテムをプレイヤーが買い取る際に支払われる買取金
  - 買取価格 = `アイテムの基本売値 × 2`
- **支出先**:
  - **食料代**: 毎日、食料として消費した分の代金を所持金から支払う（生食材2G、料理は基本売値分）
  - **宿代**: 休息中、1時間あたり `1 + 宿屋Lv` G が徴収される
  - **ポーション・スタミナポーション購入**: 派遣時に自動購入（所持金から支払われる）
- **ツケ（借金）**: 所持金が不足する場合はマイナス（ツケ）となる。ツケが溜まっている村人は戦闘能力が `0.7倍` に低下（`applySalaryDebuff`）。
- **ツケ返済**: プレイヤーは任意のタイミングで村人のツケを肩代わり返済できる。

### 仮置き場（pool）システム

村人がアイテムを獲得した際、プレイヤーのゴールドが不足していて買い取れない場合、アイテムは村人の`pool`（仮置き場）に一時保管されます。
毎時間の自動買取処理で、プレイヤーにゴールドがあれば自動的に買い取られます。

---

## 5. マーケットトレンド（経済相場）

毎日1回、アンロックされた街の中からランダムに1つ選ばれ、その街の需要アイテム1種の買取価格が `1.5〜2.5倍` に上昇します。
トレンド発生時はログに通知され、該当アイテムをその街に輸出することで通常より多くのゴールドを得られます。

## 6. 交易システム（町・馬車）

### 街（Town）

3つの交易先の街が存在します（Tier進行に応じて解放）。

- **コモレビ村** (Tier1解放): 食料・薬草・木材が特産
- **港町アイアンポート** (Tier2解放): 金属素材・武具が特産
- **魔法都市マギカ** (Tier3解放): 魔法素材・回復薬が特産

各街には**友好度**（0-1000）があり、交易を重ねると上昇。友好度が上がると売却価格にボーナスが付きます。
**投資**を行うと交易時間が短縮（投資Lvごとに10%短縮、最大50%）。

### 交易馬車（Caravan）

交易所のレベルに応じて最大5台の馬車を使用可能（Lv1: 1台, Lv2: 2台, Lv3: 3台, Lv4: 4台, Lv5: 5台）。

- **輸出**: 倉庫のアイテムを街へ運び、ゴールドと友好度を得る
- **輸入**: ゴールドを支払い、街の特産品を購入する
- **自動交易**: 取引ルールに基づき、閾値を超えた在庫を自動輸出

## 7. 目標設定と自動化システム

### ① 目標個数設定

アイテムごとに「目標個数 (Target Amount)」を設定可能。目標未達のアイテムがある場合、自動派遣・自動クラフトの優先対象となります。

### ② 自動派遣（Auto Dispatch）

待機中の村人は以下のロジックで自動派遣されます：

1. **不足アイテムの特定**: 目標個数に達していないアイテムをリストアップ
2. **職業の優先カテゴリでソート**: 職業に応じた得意カテゴリのアイテムを優先
3. **達成率でソート**: 達成率（在庫/目標）が低いアイテムを優先
4. **エリア選定**: 不足アイテムが採取・討伐できるダンジョンへ派遣
5. **ポーション・スタミナドリンク自動購入**: 派遣時に所持金から自動購入し携行

### ③ 自動クラフト（Auto Craft）

設定された目標個数に在庫が不足しており、かつ素材が揃っている場合、アイドル状態の施設で自動的にクラフトが開始されます。
職人がいれば優先割り当て、いなければ一般の村人が割り当てられます。

### ④ 自動取引（Auto Trade）

取引ルールで設定された閾値を超えた在庫アイテムを、自動で交易馬車が輸出します。

## 9. ソウルポイント (SP) と周回

### ゲームオーバー時SP計算

```typescript
// コード上の計算式
SP = Math.floor(所持ゴールド / 1000)
   + Math.floor(倉庫内全アイテムの売値合計 / 100)
   + (撃破したボス数 × 50)
   + floor(到達した最大脅威度 / 20) × 5  // 20% につき 5 SP
```

> **変更点**: 旧仕様の「到達日数 × 2」は撤廃。日数の概念は内部的に残るが（食料消費・クールダウン計算用）、SP 評価からは除外。代わりに「どれだけ危機的状況まで持ちこたえたか」を**最大到達脅威度**で評価する。20% 到達で 5 SP、80% 到達で 20 SP の上乗せ。

### 周回強化ショップ

| バフ名       | 効果                              | 1Lvあたりコスト | 最大Lv |
| :----------- | :-------------------------------- | :-------------: | :----: |
| 先祖の遺産   | 初期ゴールド +500                 |      10 SP      |   10   |
| 豊かな備蓄   | 初期食料 +100                     |      5 SP       |   10   |
| 英才教育     | 村人の獲得経験値 +10%             |      20 SP      |   5    |
| 頑強な肉体   | 全村人の初期ステータス +2         |      15 SP      |   5    |
| 効率的な建築 | 施設アップグレード素材要求量 -20% |      25 SP      |   5    |
| 値切り上手   | 転職ゴールド -10%                 |      15 SP      |   5    |

### 転生後の自動ゲーム開始

転生ボタンを押した場合、初期化後にゲームが**自動的に一時停止なしで開始**します。
手動リセットの場合は従来通り一時停止状態で開始します。

## 10. データ永続化とマスタ同期システム

Zustand の `persist` を用いて localStorage にセーブデータを保存します。
ロード時に以下のマージ処理が自動実行されます：

1. **ダンジョン情報の同期**: 探索度のみ復元し、その他のパラメータは最新のマスタデータで上書き。
2. **未建設施設のコスト同期**: `level === 0` の施設は最新のマスタデータのコストで上書き。
3. **村人の新規フィールド補完**: 古いセーブデータに不足しているフィールド（`potionCount` など）はデフォルト値で補完。
4. **脅威度フィールドの補完**: 旧セーブデータに `threatLevel` 等が無い場合、0 で初期化する破壊的変更を含むため、`saveVersion` を `3` にインクリメント（互換性確保）。

## 11. ダンジョン脅威度システム

旧来の「日数ゲームオーバー」を撤廃し、ダンジョンごとに蓄積する**脅威度（Threat Level）**を時間圧の主軸とする新システム。

### 11.1 基本概念

- 各ダンジョンは `threatLevel: 0-100` を持つ
- 時間が経過すると**未攻略ダンジョン**の脅威度が加速度的に上昇する
- プレイヤーは**ゴールドによるお布施**で脅威度を下げられる
- 脅威度が **100%** に達したダンジョンは**即ゲームオーバー**となる
- **エリアボスを討伐**すると全ダンジョンの脅威度が **0%** にリセットされ、Tier 進行と引き換えに平和が戻る

### 11.2 脅威度の上昇カーブ

未攻略ダンジョン（= `dungeon.unlockedAtTier === currentTier` かつ `bossDefeated === false` のもの）のみ、毎時以下の式で上昇する。

```
累積脅威度(t) = THREAT_BASE_RATE_PER_HOUR × t ^ THREAT_ACCELERATION_EXPONENT
```

| 定数名                         | デフォルト値 | 意味                         |
| ------------------------------ | ------------ | ---------------------------- |
| `THREAT_BASE_RATE_PER_HOUR`    | 0.1          | 1 時間あたりの基本脅威度 (%) |
| `THREAT_ACCELERATION_EXPONENT` | 1.1          | 経過時間による加速係数       |

**数値例（中程度加速カーブ）**:

| 経過時間 (h)   | 累積脅威度 (%) | 到達まで (ゲーム日) |
| -------------- | -------------- | ------------------- |
| 24 (1日)       | 3.4            | -                   |
| 72 (3日)       | 12.5           | -                   |
| 120 (5日)      | 23.5           | -                   |
| 168 (7日)      | 36.4           | -                   |
| 240 (10日)     | 59.5           | -                   |
| 288 (12日)     | 77.8           | -                   |
| **336 (14日)** | **99.0**       | **約 14 日**        |

→ Tier1 ボス討伐の推奨タイムリミットは**ゲーム内 14 日前後**。

### 11.3 攻略済みダンジョンの扱い

`dungeon.unlockedAtTier < currentTier`（既にボス討伐済み）のダンジョンは脅威度が **0%** に固定され、上昇しない。Tier 進行直後の新しいダンジョンのみ時間圧を受ける。

### 11.4 お布施（オファリング）

#### コスト計算式（指数逓増）

軽減量 `X%` のお布施コストは、Tier別の基本コスト `BASE(tier)` を元に逓増係数 `ESC` で計算する：

```
cost(X) = BASE(tier) × (ESC^X − 1) / (ESC − 1)
```

| 定数名                       | デフォルト値                | 意味                                        |
| ---------------------------- | --------------------------- | ------------------------------------------- |
| `OFFERING_BASE_GOLD_BY_TIER` | `[0, 2, 10, 50, 250, 1000]` | Tier別の基本コスト (G/%)。indexがtierに対応 |
| `OFFERING_ESCALATION_FACTOR` | `1.08`                      | 軽減量に対する逓増係数                      |

**数値例（Tier1, BASE=2, ESC=1.08）**:

| 軽減量 X% | コスト (G) | 1% 平均コスト |
| --------- | ---------- | ------------- |
| 5%        | 12         | 2.4           |
| 10%       | 29         | 2.9           |
| 20%       | 92         | 4.6           |
| 30%       | 227        | 7.6           |
| 50%       | 1,148      | 23.0          |
| 90%       | 25,448     | 282.8         |

**数値例（Tier5, BASE=1000, ESC=1.08）**:

| 軽減量 X% | コスト (G) |
| --------- | ---------- |
| 10%       | 14,487     |
| 30%       | 113,284    |
| 50%       | 573,771    |

> 注: 上記の数値は `calculateOfferingCost(X, tier) = Math.ceil(BASE × (ESC^X − 1) / (ESC − 1))` を
> 実行して算出した実装値です。Tier 進行に従ってコストが急激に増加するため、
> 終盤 Tier での全額リセットはお布施では事実上不可能（ボス討伐が最善）。

**設計意図**: 序盤の 5〜10% 軽減は気軽に払えるが、終盤に 90% を払うと Tier に応じて数万 G と高額。**全額リセットはお布施では割に合わず**、結局ボス討伐が最善という圧を維持する。

#### お布施の実行

プレイヤーが任意のタイミングで `offering(dungeonId, percentToReduce)` アクションを呼び出せる。バリデーション:

- `1 ≤ percentToReduce ≤ 100` の整数（実装上 100% まで許容）
- `gold >= cost(percentToReduce)`
- `dungeon.threatLevel >= percentToReduce`（現在値以上に下げられない）
- `dungeon.threatLevel > 0`（既に 0% の場合は不要）

成功時: `gold -= cost`, `dungeon.threatLevel -= actualReduction`（actualReduction は threatLevel と percentToReduce の小さい方）、ログに「【お布施】…奉納し、脅威度を X% 低下させました。」を記録。

### 11.5 ゲームオーバー（脅威度MAX到達）

> **変更点**: 旧仕様の「襲来イベント（猶予6時間・被害発生・3回累積でゲームオーバー）」は
> 撤廃済み。実装上は **脅威度が 100% に達した時点で即ゲームオーバー** となる。
> `raidFailureCount`, `raidEvent`, `RaidWarningModal` 等の旧フィールド/UI は削除されている
> （persistence.ts v4→v5 migration で完全に除去）。

#### 発動条件

```
threatLevel >= THREAT_MAX (=100)
```

を満たした未攻略ダンジョンがあるとき、その時点でゲームオーバーとなる。

| 定数名                         | デフォルト値 | 意味                         |
| ------------------------------ | ------------ | ---------------------------- |
| `THREAT_MAX`                   | 100          | ゲームオーバー閾値 (%)       |
| `THREAT_BASE_RATE_PER_HOUR`    | 0.1          | 1 時間あたりの基本脅威度 (%) |
| `THREAT_ACCELERATION_EXPONENT` | 1.1          | 経過時間による加速係数       |

#### ゲームオーバー時の処理

- `acc.gameOver = true`
- `acc.gameOverReason = "脅威度"`
- `acc.isPaused = true`
- ログ: 「【ゲームオーバー】{ダンジョン名} の脅威度が限界 (100%) に達しました。村は壊滅しました…」

襲来猶予や被害処理のロジックは存在しない。即座にゲームオーバー画面へ遷移する。

### 11.6 ボス討伐時のリセット

`processBossBattle` 内で `currentHp <= 0` を検知した時点で、`dungeons` を以下のように更新:

```typescript
const resetDungeons = dungeons.map((d) => ({
  ...d,
  threatLevel: 0,
}));
// activeBoss を null に
```
